# 远程访问 UI 教程

这篇教程按“纯界面主路径”带你完成远程访问配置。

## 开始前准备

请先确认：

- 本地 NextClaw UI 已经可以正常打开
- 这台机器已经有你自己的外部接入方式
- 如果你不用默认平台地址，已经明确要使用的 API Base

## 第一步：打开页面

在 UI 中进入：

- `设置`
- `远程访问`

你会看到账号、设备、服务、诊断四个区域都在一个页面里。

## 第二步：在浏览器里完成授权

在 `平台账号` 区域：

1. 如果平台地址需要改，优先先在 `设备配置` 里改好
2. 点击 `前往浏览器授权`
3. 在新打开的平台页面里完成登录
4. 如果还没有账号，直接在那个页面里切到注册并完成授权
5. 回到本地 UI，等待页面自动完成登录态接入

如果你刚刚执行过 `退出登录`，重登时也还是走这同一个按钮，不需要再借助 CLI。

## 第三步：保存设备设置

在 `设备配置` 区域：

1. 打开 `启用远程访问`
2. 设置一个容易识别的设备名
3. 如有需要再覆盖平台 API Base
4. 点击 `保存设置`

## 第四步：启动或重启托管服务

在 `后台服务` 区域：

- 如果服务还没启动，点击 `启动服务`
- 如果你刚修改了远程设置，点击 `重启服务`

如果当前页面本身就是由这个托管服务提供的，那么重启时页面短暂断开是正常现象。

## 第五步：运行诊断

点击 `运行诊断`，重点确认这些检查项通过：

- `remote-enabled`
- `platform-token`
- `local-ui`
- `service-runtime`

## 第六步：换另一台设备验证

在另一台设备上：

1. 打开你的外部入口地址
2. 确认 NextClaw UI 能正常加载
3. 打开 `远程访问` 页面，确认连接器状态为已连接
